相交链表

可以用hash表存储遍历过的节点,需要遍历两次

这里用双指针

把A链表 和B链表接起来

把链表分为以下几种

A + 相遇点 + 共同路径

B + 相遇点 + 共同路径

我们分别在A后加B B后加A

A + 相遇点 + 共同路径 + B + xx

B + 相遇点 + 共同路径 + A + xx

这个xx的部分,就是相遇点,而前面的部分,是一样长的,也就是说遍历完一遍后再去分别遍历另一条,即可找到相遇点

如果没有相遇点, 就会变成

A + B

B + A

同样会同时到达最后的nil 然后退出循环

二叉树的最近公共祖先

递归思路,寻找左右子树,如果左子树为nil,说明都在右子树里,反之都在左子树

如果左右子树都有值,当前点就是最近公共祖先
