Interface的结构里有data和type

当底层是值类型的时候,直接存在data里,当底层是应用类型的时候,data区域保存的仅仅是个指针,进行类型断言的是指针指向的那个地址

接口实现

一个类型只要实现了定义接口要求的方法,就看做是实现了该接口

接口底层是iface或者eface

itab是类型表

接口底层的实现过程

接口的零值与nil判断,分为类型和值,同为nil的时候才为nil

局部变量分配在栈上还是分配在堆上

如果变量的生命周期局限于函数作用域,不会逃逸到函数外,那就分配在栈上

如果局部变量的生命周期超出函数作用域, 编译器就将变量分配在堆上, 这就是逃逸分析

栈和堆

栈是线程私有的,分配和释放由编译器管理,分配速度快

堆事共享的,分配和释放由GC管理,动态分配和长生命周期,但是性能较慢

数组和切片区别

见过很多次的问题了

数组的大小固定,内存连续,传递时进行值传递,并且数组长度是数组类型的一部分

切片大小是动态的,支持动态扩展

数据结构里有三部分: 底层数组的指针, 切片长度, 切片容量

切片传递的时候传递的是引用,切片扩容传递的时候会创建新的底层数组,切片的动态扩容过程我在Go语言设计与思考中详细分析过了

其中字符串数组比较特殊,是个只读数组

切片扩容

在1.18以前,容量小于1024下进行2倍扩容,大于1024进行1.25倍扩容

在1.18以后,小于256进行2倍扩容,大于256进行newcap += (newcap + 3*threshold) >> 2公式扩容,在newcap为256的时候,正好为两倍,当newcap无限大,正好接近于1.25倍

扩容计算后需要内存对齐,内存对齐数组在Go语言设计与思考一文中有提及

输出结果不是两个5,而是5和6

很显然,扩容5个元素,需要40个字节,内存对齐,最靠近的元素是上方的48个字节,所以cap为48/8=6个字节

init函数在何时执行

执行顺序

  1. 根据import依赖关系做一个拓扑排序
  2. 逐个执行它们的包级初始化和init()
  3. 程序入口

在main.main函数执行之前所有代码都运行在同一个Goruntine,主系统线程

如果某个init()函数启动了新的Goruntine,那么就是和主系统线程并发执行了

rune byte

rune就是int32,占用4个字节,统计一个字符串的长度的时候,如果有中文和英文,中文会是两个字符长度,转换成rune数组再判断长度就是正常的字符数量了,因为rune可以用来表示unicode

byte就是uint8,占用一个字节,字符串实际上是byte切片

深拷贝和浅拷贝

深拷贝复制所有字段

浅拷贝只是复制了引用或者指针

默认浅拷贝类型有slice,map, 指针, func, chan, Interface

copy不会递归复制指针所指向的内容,所以算是浅拷贝,要实现深拷贝,要遍历整个数据结构(如果有嵌套结构体的)

map的值不可寻址

无法取得map的地址

如果map中的值是基本类型,直接修改就行

如果是结构体类型,就得把map的value定义为引用类型或者创建临时变量

创建临时变量的就不写了

引用类型和指针有什么不同

go语言的引用类型在数据结构里有一个指针

指针类型的变量本质就是一个内存地址

判断map中是否包含某个key

map如何实现两种get操作

一种仅返回值,一种还返回是否存在的bool值

如果map是nil,返回的值分别是 0, false

map的key是无序的

为什么map是无序的呢,在本站的另一篇文章,go语言的设计与思考中有提到go里的map的实现,其中map会发生扩容,如果按照顺序确定,那么在实现扩容时,就会打乱之前的位置

另外hash表的实现不维持键之间的插入排序顺序,检索更灵活,不需要花费额外代价去维护map

如果需要实现有序

Go 不支持默认参数和可选参数 (吗)

所有函数的参数必须显示指定

但是有一个妥协办法,就是可变参数 func foo(opts ...string),或者定义结构体,把要传入的东西全部放在结构体里,可以给结构体默认值

defer执行顺序

参考栈,先进的后出,后进的先执行

注意点

  1. defer无法跨 goruntine捕获panic
  2. defer语句下的函数参数在defer语句执行时被立即计算,但是可以写闭包
  3. defer可以嵌套

defer一般用于资源管理,错误处理,传递参数等等

  1. json编解码
  2. 数据库orm
  3. validator
  4. xml编码解码

字符串转成byte数组时会发生内存拷贝吗

字节数组可改变,字符串不可变,显然发生了内存拷贝

但是如果使用unsafe包对原地址转换呢

这种情况下不会发生拷贝,但是不能修改[]byte的值

make和new的区别

new会为一种类型分配内存,但是它什么都不初始化,只是返回一个指向零值的指针

make只用于分配和初始化特定类型的数据结构,返回的是一个已初始化好的值

map扩容机制

增量扩容

键值对数量大于8 且 大于桶数组的6.5倍

此时桶数量翻倍

等量扩容

溢出桶超过一定数量

频繁插入元素又删除元素,导致溢出桶增多,键值对总数不高,但是key比较分散,桶数量不变

B < 16时, 溢出桶的最大数量限制为2的B次方, 超过即扩容

B >= 16时, 设定上限为2的15次方

B是当前map的桶数量的对数,为什么这样设计在本站Go语言设计与思考中亦有提及

扩容过程

确定扩容后,进入渐进式扩容迁移状态,不会一次性搬迁到新的桶中,每次最多只会搬迁2个桶

怎么从panic恢复

如果panic路径跨越函数调用边界,捕获需要在每个层次都添加recover

字符串和byte切片之间的零拷贝

使用unsafe

但是在1.22版本,已经被优化为零拷贝了

java和go的区别

性能

Go默认性能比Java好

goruntine默认占用内存很小,占用资源小使得他在云原生上有得天独厚的优势(你说k8s, Docker是什么写的来着-- +)

Go使用的goruntine避免了内核态和用户态切换的成本

编译部署

Java通过虚拟机编译,使用JVM跨平台编译

Go针对不同的平台,编译对应的机器码

访问权限

Java的public private protected等啊

Go里用大小写确定访问权限

异常处理

Go里是痛苦的Error

相比之下 Java的错误Error和Exception
Error由JVM抛出交给系统处理,一些内存溢出吗,JVM非正常运行等使程序无法启动的异常;Exception是程序运行中遇见的空指针 越界等,可以被捕获后,不影响程序运行

继承与多态

Java里通过extends不支持多继承
Go里的struct里直接写另一个struct的定义就好,可以多继承

Java多态必须满足继承,重写,向上转型;不同实体类型对接口值方法的调用就是多态

而Go里只需要某个类型实现了接口中的方法就相当于实现了该接口

值传递和指针传递

Java没有显示的指针,Java数组是引用,Go是值
Go里切片 channel func map 指针等是引用

并发

Java借助共享内存作为线程通信媒介,会有线程不安全(通过加锁,原子类等可解决问题)
Go使用channel,使用GMP并发模型,多个goruntine通过channel来通信

gc

Java通过JVM虚拟机的分代收集算法完成GC,Golang使用并发的三色标记算法来GC
